# CS255-System-Analysis-and-Design
# CS-255 Portfolio Reflection

## DriverPass Project Summary

The DriverPass project involved designing a system for DriverPass, a company that wanted to help customers prepare for their driver's license exams. The client wanted a web-based system that allowed users to create accounts, schedule driving lessons, take online practice tests, manage appointments, and make payments. The system also included features for administrators, instructors, and secretaries to manage customers, schedules, and business operations.

## What I Did Well

One of the things I did particularly well was gathering and organizing the client's business requirements into a clear and detailed document. I also created UML diagrams and a system design that matched the client's needs while considering different user roles and system functionality.

## What I Would Revise

If I could revise one part of my work, I would improve the system design document by adding more detail to the UML diagrams and expanding the explanations of how each system component interacts. This would make the design easier for developers and stakeholders to understand.

## Interpreting User Needs

I interpreted the user's needs by carefully reviewing the client's interview and identifying both functional and nonfunctional requirements. I used those requirements to design a system that supports scheduling, online learning, secure user accounts, and administrative tasks. Considering user needs is important because a system is only successful if it solves the customer's problems and provides an easy and reliable experience for its users.

## My Approach to Software Design

When designing software, I begin by understanding the client's goals and gathering all necessary requirements. I then organize the information into diagrams and documentation before designing the overall system architecture. In the future, I will continue using techniques such as requirements analysis, UML diagrams, process modeling, and user-centered design to create systems that are organized, efficient, and meet customer expectations.
